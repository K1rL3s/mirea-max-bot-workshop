## Боты

### ТГ

Можно создать 20 ботов без премимум и 40 с премиумом

Можно выбрать имя, описание, описание при запуске, картинку при запуске, аватарку, команды, политику конфиденциальности. \
Можно передать бота, настроить платёжку звёздами, инлайн режим, бизнес-мод-секретарь, ДОПОЛНИТЬ

Плюсы:
- Можно скачать медиа по `file_id`
- Чтобы отправлять и качать медиа, надо хранить только `file_id`
- Настройка списка команд для всех юзеров или конкретным юзерам
- Есть ивенты на промоут и демоут (повышение до админа и разжалование) бота

Минусы:
- Нельзя загрузить медиа, не отправив его кому-то
- API скорее по кайфу, чем REST

Особенности:
- Документация по `/bots/api` - простые HTML таблицы
- Есть как и `HTTP API`, так и `MTProto` со своим `Type Language`. По сути http api - прослойка между http и mtproto
- Работа с альбомами (несколькими медиа в одном сообщении) идёт как работа с несколькими сообщениями, одно сообщение = одно медиа
- У фото несколько размером
- Кнопки на локацию и контакт реплайные
- Telegram бот создаётся сразу через BotFather, без отдельной модерации. Есть inline mode, payments, web apps, attachment menu и command scopes.
- `file_id` у Telegram работает как кэш внутри конкретного бота, поэтому медиа можно переиспользовать без повторной загрузки, но между ботами этот идентификатор не переносится.


### Макс

#### Создание

Для организаций доступно 5 ботов, СЗ - 2 (больше через обращение в поддержку, госам точно можно больше 5).

При создании бота можно выбрать аватарку, имя, телефон компании, сайт компании, описание. \
Организацям выдаётся юзернейм `id<ИНН>_<N>_bot`, СЗ - `se<id>_<N>_bot`, госам - любое с `bot` в конце. Первый бот выдаётся без `_<N>_`.

После создания бота он отправляется на модерацию (до 48 часов). Условный живой человек проверяет, что бот не нарушает [правила](https://dev.max.ru/docs/legal/rules). \
Если всё хорошо, то занимает 5-10 минут. Если дольше, то очень вероятно, что придёт отказ

#### API

Плюсы:
- Можно одновременно запустить несколько лонг-поллингов или вебхуков. На каждый из них будут приходить апдейты
- Максимум 12 медиа-аттачментов (фото, видео) в одном сообщении, против 10 у ТГ
- Есть шаринг текста по диплинкам (`https://max.ru/:share?text=Привет`) (`https://t.me/share/url?url={url}&text={text}`)
- Можно загружать медиа, не отправляя его
- Есть ссылка на медиа
- API в целом REST-подобный

Минусы:
- Бот не может удалить сообщение юзера в диалоге. API отдаёт 200, но на клиенте ничего не происходит.
- Бот не может изменить информацию о себе. (Может, но неофициально, криво-косо, и не гарантированно)
- Есть ивенты на добавление и удаление бота из чата/канала, но нет ивента на повышение до админа. Без админки бот не получает ивенты из чата/канала
- Бот не может отправлять видео-кружки (может, но по токену, и кружок должен юзер отправить боту).
- Бот не может отправить reply-клавиатуру, которая прикреплена к низу чата (в Максе нет понятия как reply-клавиатура). Есть MessageButton'ы, по котором юзер отправляет текст в чат.
- Странное поведение списка команд бота. Не настраивается через бизнес-аккаунт, но у некоторых ботов есть
- Нет inline режима
- Нельзя скачать медиа по `token`, только по ссылке (есть гет файла для видео, но не для фото и файлов)
- Callback-кнопки не блокируются на клиенте после нажатия, можно ими флудить
- В веб версии не отображаются ответы на callback-кнопки
- В веб версии не отображаются некоторые стили форматирования
- Нет стилей callback-кнопок
- callback.answer работает только на iOS
- Список команд на iOS доступен всегда, на андроиде после ввода слэша (`/`), на остальных платформах нету вообще

Особенности:
- Фото, видео, файлы, аудио, геолокация, клавиатура итп отправляются как `attachment` (вложения)
  - Из-за этого можно в одном сообщении отправить и клавиатуру, и много фоток
  - Альбомы приходят одним сообщением
- Чтобы отправлять и качать медиа, надо хранить `token` и `url`
- Документация по `/docs-api` генерируется из сваггера
- У фото один размер - максимальный
- Кнопки на локацию и контакт инлайновые
- Клавиатура в MAX описана как inline-клавиатура внутри `attachments`, а не как отдельный reply-контур. В одной клавиатуре может быть до 210 кнопок, до 30 рядов и до 7 кнопок в ряду.
- В MAX вебхук и long polling нельзя держать одновременно на одном токене. Либо только вебхуки, либо только лонгполлы
